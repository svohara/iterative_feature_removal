compute_annotations_per_iteration(self, depth=5, saveas=None, bf_thresh=0.0, num_iterations=10, homologs=False)

source code 

Given a file with iterative gene removal data, this function will plot a bar chart of the GO annotations at a given level (depth) for each iteration.

Parameters:
  • depth - The GO ontology depth to use.
  • saveas - Provide a filename to save the GO annotation results. This will be a python pickle file. Specify None to avoid saving the results to a file.
  • bf_thresh - The bayes factor threshold. An annotation must have at least this bayes factor before being counted in the results.
  • num_iterations - The number of iterations to process
  • homologs - Select whether to include gene homologs in the analysis.

_plot_GO_bar_chart(self, iteration, title=None, annotation_set=None, xlim=None, suppress_ylab=False, suppress_desc=False, desc_filter=True, color='lavender', show=False)

source code 

Creates the bar chart visualization of the dat produced by compute_GO_annotations_at_depth() function.

Parameters:
  • iteration - which iteration of the results to show, zero-based index
  • title - Used in the plot title
  • annotation_set - Optional. If provided, then the bar chart will show values exactly for this list of annotation ids. If dat does not include the annotation id, then a zero-count is shown. If dat has an annotation id not in this set, it will not be output. If None, then the annotation set is the set of annotations in dat. This is useful if you are using this function to plot subfigs and you'd like to have the same set of y-labels for each. If provided, format is a list of tuples [ (annotation_id, desc)]
  • xlim - Optional. If provided, the x-axis will be limited to [start,end] values.
  • suppress_ylab - If true, only ticks will be rendered on y-axis labels, no text.
  • suppress_desc - If true, the text descriptions for the GO annotation will not be drawn
  • desc_filter - If suppress_desc is FALSE and this is TRUE, then descriptions will be drawn to the bar chart only for those non-zero annotations.
  • show - If False, the plot won't be shown. This is useful when this function is called in a loop to generate subplots in a master figure.

An alternative way to visualize the annotations per iterative via a heat-map like construct. The rows are the GO annotations and the columns are iterations. The color intensity indicates how much a given annotation was present in a given iteration
